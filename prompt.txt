CORE GEOMETRY CONCEPT

The algorithm must start from a pyramidal configuration with four triangular faces.

Use a square pyramid as the default seed:

- Coordinate system: Three.js Y-up.
- Base square centered at origin on y = 0.
- Apex on the vertical Y axis.
- Four triangular side faces:
  face 0: apex, base corner 0, base corner 1
  face 1: apex, base corner 1, base corner 2
  face 2: apex, base corner 2, base corner 3
  face 3: apex, base corner 3, base corner 0

The base cap should be optional. Default: no base cap, so the form reads as a crystalline shell or architectural mass.

Each initial triangle should act as a recursive field. The three edges of every triangle are treated as “axes.” Each axis should be broken using a Koch curve-like operation. The recursion should happen on triangular faces, not on a heightfield, voxel grid, sphere, or terrain.

TRIANGULAR KOCH-LIKE RECURSION

Implement a recursive function:

generateFractalTriangle(
  a: THREE.Vector3,
  b: THREE.Vector3,
  c: THREE.Vector3,
  level: number,
  params: FractalParams,
  rng: SeededRandom
): Triangle[]

For each triangular face:

1. Compute face properties:
   - centroid g
   - outward normal n
   - average edge length
   - local 2D coordinate frame for the triangle

2. For each edge of the triangle, apply a Koch-like break:
   For edge P to Q:
   - p1 = lerp(P, Q, splitRatio)
   - p2 = lerp(P, Q, 1.0 - splitRatio)
   - middle segment = p2 - p1
   - Create a Koch peak k by rotating the middle segment in the local triangle plane by kochAngle.
   - Choose the peak direction that points away from the triangle centroid in local 2D coordinates.
   - Lift the peak in 3D using:
     k3D = kPlanar
           + faceNormal * normalExtrusion * edgeLength * levelDecay
           + globalUp * verticalBias * edgeLength * levelDecay
           + radialOut * buttressBias * edgeLength * levelDecay
   - Add controlled random variation to angle, height, split, and radial direction, using a deterministic seeded random generator.

The boundary of the modified triangle should become a 12-point polygon:

A,
AB_p1,
AB_peak,
AB_p2,
B,
BC_p1,
BC_peak,
BC_p2,
C,
CA_p1,
CA_peak,
CA_p2

3. Convert this 12-point polygon into triangular facets.

Provide two triangulation modes:

A. "Flat Patch Mode"
   - Use earcut or another triangulation method in local 2D.
   - Map triangulated vertices back into 3D.
   - This creates a Koch-broken triangular surface.

B. "Spire Fan Mode"
   - Default mode.
   - Compute a raised central point:
     center = centroid
              + faceNormal * centralLift * avgEdgeLength * levelDecay
              + globalUp * centralSpireBias * avgEdgeLength * levelDecay
              + radialOut * centralButtressBias * avgEdgeLength * levelDecay
   - Create fan triangles from each boundary edge to this raised center.
   - This creates Gothic spires, folded facets, and crystalline peaks.

4. Recurse on the generated triangles until level == recursionDepth.

5. Stop recursion if:
   - level reaches recursionDepth
   - triangle area is below minArea
   - total triangle count exceeds maxTriangles

IMPORTANT: The recursion should visually read as a Koch-like breaking of triangular axes, then an architectural extrusion/folding of those broken axes into faceted mass.

AXIS ATTRACTOR

Because the design should feel architectural and not purely chaotic, implement an optional axis-attractor system.

Define major axes:
- The vertical central axis from base center to apex.
- Four diagonal/ridge axes from each base corner to the apex.
- Four face axes from each base edge midpoint to the apex.

When creating new peak points, allow the point to be slightly attracted toward the nearest major axis. Use axisAttractorStrength as the blend value.

Pseudo logic:

function attractToNearestAxis(point, axes, strength) {
  nearest = closestPointOnAnyAxis(point, axes)
  return lerp(point, nearest, strength)
}

Only apply this after the Koch peak and extrusion have been computed. This should make the recursive spires feel aligned with a hidden Gothic/crystalline skeleton.

SYMMETRY

Implement symmetryMode.

If symmetryMode is true:
- The four initial pyramid faces should use the same random sequence at each recursion level.
- This preserves a four-sided architectural organization while still allowing local variation.

If symmetryMode is false:
- Each face can use independent seeded randomness.
- This creates more turbulent German Expressionist asymmetry.

- Use BufferGeometry.

- Show edge lines using EdgesGeometry or LineSegments.
- Display triangle count in the UI.
- Warn or clamp when recursionDepth and maxTriangles would produce too many triangles.
